# Status da base

- Backend Django modular.
- Postgres configurado.
- Admin nativo.
- API REST.
- Clientes.
- Produtos e itens.
- Estoque.
- Vendas com pedidos e itens.
- Financeiro básico.
- Agentes Lívia/Atlas/Policy Guard.
- Módulo de páginas HTML.

## Não incluso de propósito

- Template visual fixo.
- Vue/React.
- Loja virtual completa.
- Gateway de pagamento.
- LLM externo plugado.
- Envio automático irrestrito de e-mails.

## Próximo passo natural

Escolher o primeiro nicho do Smart System, integrar HTML específico e criar módulos de operação específicos, por exemplo OS, agenda, contratos ou e-commerce.

## Atualização — Orçamentos, vistorias e relatórios

A base foi expandida para atender empresas de serviço que precisam iniciar orçamento em campo:

- cadastro de imagens de produto no catálogo;
- módulo `estimates` para orçamento editável;
- itens de orçamento com cálculo de subtotal/total;
- fotos de vistoria vinculadas ao orçamento;
- medições de campo vinculadas ao orçamento;
- geração de mensagem de primeiro contato;
- UI web responsiva para dashboard de orçamentos;
- UI mobile para vistoria com captura de foto pela câmera do celular;
- módulo `service_reports` para relatórios de serviço;
- fotos antes/durante/depois e evidências técnicas em relatórios;
- CSS base forte e neutro em `static/css/smart_system_ui.css`.

A base continua sem template comercial fixo. A UI adicionada é operacional e neutra para ser substituída/adaptada conforme o nicho.
